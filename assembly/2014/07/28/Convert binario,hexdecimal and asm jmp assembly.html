O que é Hexadecimal ?

O sistema hexadecimal é um sistema de numeração posicional que representa os números em base 16, portanto empregando 16 símbolos.(wiki) Hexadecimal. Hexadecimal, hex ou como é mais conhecido, é uma base de 16 sistema de numeração. Base de dados de 16 o que significa que é composto por 16 números:0-9 e AF. Cada um destes números (AF = 10-16) tem um valor de 4 bitse também são chamados de petiscos. Ao representar um número hexadecimal, um iria escrever um "0x" antes de o conjunto de bits real. 0x é simplesmente uma opção de venda tag antes de um número hexadecimal para deixar os programadores sabem que ele é, de fato, hex. Ao escrever hex, você não vai precisar usar esse prefixo.

"\x41" <-- A
"\x42" <-- B
"\x43" <-- C

E já que toquei no Assunto RAM vamos conhece-la melhor agora: O que é memória RAM? RAM (Random Access Memory) é basicamente memória e o processo de acesso a ele. O termo "Random Access Memory" foi dado a esta unidade de memória, porque ao executar um comando, a CPU não tem que percorrer toda a memória em PC até encontrar o endereço certo. Como a RAM funciona?Quando um comando é emitido e a memória é retirado do arquivo, ele deve primeiro passar por o que é chamado um "vector". Um vetor é uma "porta de entrada" ou um "setor" de RAM onde o endereço da função é armazenado com outros de sua própria espécie. Um exemplo de um vector seria algo como isto:

8c0000b4-8c00ffff

Isto significa que todos os "addressii" (hehe) que estão entre estes valores são armazenados em que sector da memória RAM. Um vetor age como um gateway em que, primeiro, passar por um vetor para chegar ao endereço. Seu programa tem média probley cerca de 30 a 40 vetores principais, seccionando fora de inicialização até a saída. Sabendo o vetor de um addy ou uma função irá reduzir significativamente sua dor de cabeça quando você começar a procurar por ele.

O Sistema Binário:
Aqui está um exemplo simples de binário:
10101

Vamos pensar no exemplo acima slots como vazios:
_____

Primeiro, você lê binário da direita para a esquerda. É apenas a maneira como ele foi projetado. O primeiro slot da direita representa um valor de um, o segundo da direita um valor de dois, o terceiro da direita um valor de quatro, o quarto da direita um valor de oito , o quinto da direita um valor de dez , e o ciclo continua por múltiplos de 2. Isso nunca vai mudar.Ao colocar um 1 ou um 0 você quer dizer que você quer valor correspondente que está ligado a esse slot ou não. A 1 significa sim, e um 0 significa que não há. Por exemplo, colocando um zero no primeiro slot da direita, mas a 1 no segundo slot da direita significa que você quer um dois, mas não um:

___ 1 0

Como tal, o número acima é igual a um valor decimal de dois.

Como exemplo, digamos que você deseja representar oito na forma binária. Bem, pensando sobre os slots, você quer o primeiro slot para ser 0, pois você não quer um, você quer que o segundo slot para também ser 0, porque você não quer um dois, você quer o terceiro slot para também para ser 0 porque você não quer que um quatro, mas você quer o quinto slot de ser 1, porque você quer um valor de oito. Como tal, de oito em forma binária é: 1 0 0 0.Agora, é importante notar que a quantidade de zeros que precedem o primeiro valor de um a partir da esquerda não é importante. Assim, por exemplo: 1 0 0 0 é o mesmo que 0 0 0 1 0 0 0 (1000 = 000100) Para obtê-lo esclarecido, aqui está outro exemplo: 1 0 é o mesmo que 1. Aqui está um exemplo de uma mensagem "escondida" no texto binário:

0100100001100101011011000110110001101111

Tendo conjuntos de 8 dígitos para o seu equivalente binário é mais do que suficiente para representar todas estas carta. todas as sequências que representam texto como no exemplo acima são separadas em pedaços de 8 para simplificar(lembre-se sempre em 8):

01001000 01100101 01101100 01101100 01101111

Ok, então a nossa mensagem exemplo foi separado em oito sequências de dígitos. O valor decimal para cada uma dessas cordas no exemplo foi calculado para você.

01001000 = 72
01100101 = 101
01101100 = 108
01101100 = 108
01101111 = 111

O resultado foi 72.101.108.108.111. Agora, Sabe a tabela ASCII citada acima ?. Ele corresponde essencialmente aos números binários de outrora às letras / símbolos / números equivalentes. Mas desde que encontramos os valores decimais destas cadeias binárias, podemos usar um atalho importante. Ao pressionar ALT + [Número], você receberá o equivalente ASCII desse número. Por exemplo, ao pressionar a tecla ALT e, em seguida, (mantendo-a para baixo) os números de 72 em qualquer editor de texto, você vai ter a correspondente "H" para aparecer.

Vamos fazer por toda a mensagem exemplo:

72 = H
101 = e
108 = l
108 = l
111 = o

Então, toda a mensagem "escondida" se traduz em "Olá" :D simples não.
